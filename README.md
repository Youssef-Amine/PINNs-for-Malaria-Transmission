# Physics-Informed Neural Networks for Malaria Transmission Dynamics

This repository contains the notebooks and reference data used in the article on malaria transmission dynamics, mathematical analysis, numerical simulation, and deep-learning approximation with Physics-Informed Neural Networks (PINNs).

The workflow follows the same scientific logic as the paper:
1. analyze the analytical stability of the system;
2. generate reference PDE solutions with the numerical solver;
3. solve the forward problem with a PINN;
4. solve the inverse problem with two PINN settings.

The file `malaria_simulation_results.npz` stores the reference simulation data generated by the numerical solver and is required by the PINN notebooks.

## Repository structure

### 1. `Stability_Analysis.ipynb`
This notebook studies the **analytical stability** of the malaria transmission subsystem through the reaction matrix.

It includes:
- a global parameter sweep over \((\Lambda, \phi, f_T)\);
- stability classification based on the sign of the spectral abscissa;
- representative stable, marginal, and unstable regimes;
- interactive graphics and dashboards for manual inspection of parameter combinations;
- article-oriented 2D and 3D visualizations, including phase diagrams, stability maps, spectral diagnostics, and critical-threshold plots.

In short, this notebook is used to **simulate and visualize the analytical stability analysis** presented in the article.

### 2. `Numerical_solver.ipynb`
This notebook implements the **IMEX-Euler numerical solver** for the malaria transmission PDE system.

It includes:
- the space-time discretization of the PDE model;
- the construction of transport, diffusion, and reaction operators;
- the generation of the reference numerical solution on the \((t,a,\xi)\) domain;
- the export of simulation outputs into `malaria_simulation_results.npz`;
- diagnostic visualizations of the five compartments \(S, D, T, A, P\).

This notebook is the **data-generation stage** of the pipeline. Its outputs are used as reference data for both the forward and inverse PINN experiments.

### 3. `PINN_Forward.ipynb`
This notebook solves the **forward problem** with a Physics-Informed Neural Network.

It includes:
- loading the reference NPZ file generated by the numerical solver;
- a Fourier-feature MLP architecture;
- PDE residual construction;
- physics-based, data-based, and boundary-condition losses;
- training and evaluation of the forward PINN;
- visual comparisons between numerical and PINN solutions.

Its purpose is to learn the malaria PDE solution from the governing equations and reference data.

### 4. `PINN_Inverse_5Parameters.ipynb`
This notebook solves the **inverse problem with five unknown parameters**:
\[
(d, v, \Lambda, \phi, f_T).
\]

It includes:
- robust data-only OLS initialization;
- adaptive and structure-aware inverse losses;
- joint estimation of diffusion, advection, and reaction parameters;
- uncertainty weighting, curvature-aware collocation, and identifiability diagnostics.

This was the **first inverse setting investigated** in the study. It is the most ambitious formulation, since all five parameters are learned simultaneously.

### 5. `PINN_Inverse_4Parameters.ipynb`
This notebook solves the **inverse problem with four unknown parameters** while keeping diffusion fixed.

The learned parameters are:
\[
(v, \Lambda, \phi, f_T),
\]
with \(d\) fixed.

It includes:
- a stabilized inverse formulation derived from the five-parameter study;
- OLS projection and EMA-based structural guidance;
- ratio-consistency losses and targeted stabilization of parameter estimation;
- improved robustness for the inverse identification task.

As explained in the article, the **five-parameter inverse problem was studied first**, and the four-parameter version was then introduced as a more stable and better-identified alternative.

## Recommended execution order

For full reproducibility, the notebooks should be used in the following order:

1. `Stability_Analysis.ipynb`
2. `Numerical_solver.ipynb`
3. `PINN_Forward.ipynb`
4. `PINN_Inverse_5Parameters.ipynb`
5. `PINN_Inverse_4Parameters.ipynb`

## Data file

### `malaria_simulation_results.npz`
This file contains the reference numerical results produced by `Numerical_solver.ipynb`. It is required by:
- `PINN_Forward.ipynb`
- `PINN_Inverse_5Parameters.ipynb`
- `PINN_Inverse_4Parameters.ipynb`

Typical arrays stored in this file include:
- `U_snaps`: snapshots of the five-compartment solution;
- `times`: saved time instants;
- `age_grid`: age discretization;
- `xi_grid`: exposure discretization;
- `mu_true`: reference transport parameters used in simulation.

## Notes

- The notebooks were organized for article reproducibility and readability.
- Some cells were designed for Google Colab usage, especially for uploading/downloading the NPZ file.
- The PINN notebooks depend on the numerical data generated by the solver; they should not be run before `malaria_simulation_results.npz` is available.
